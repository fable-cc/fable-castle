# 危机不是用来渡过的，是用来升级的——压力决策的底层算法

> 顺境看谁跑得快。逆境看谁不倒下。绝境看谁从废墟里站起来，比倒下前更强。

## 普普通通的危机观

大多数人对危机的态度：**"赶紧撑过去，恢复正常。"**

这句话本身就是问题。

什么叫"恢复正常"？正常 = 危机之前的状态。那个状态在危机面前被打穿了，说明它本来就不够强。你回到那个状态——等于回到一个已被证明脆弱的系统里。

**回到危机前的"正常"，等于回到下一次危机前的"脆弱"。**

真正的危机应对——不是恢复原状，是**用危机升级系统**。

## OODA 循环：危机的底层算法

空军飞行员约翰·博伊德发明了 OODA 循环——Observe（观察）、Orient（定向）、Decide（决定）、Act（行动）。

这不是一个一次性流程。是一个**越转越快的闭环**。

| 阶段 | 危机中做什么 | 普通人怎么做（错的） |
|------|------------|-------------------|
| 👁️ Observe | 收集信息，搞清楚到底发生了什么 | 刷新闻·恐慌·信息过载 |
| 🧭 Orient | 用自己的框架分析——"这对我意味着什么" | 跟着别人的判断走 |
| 🎯 Decide | 做出一个具体决策——哪怕不完美 | 犹豫·等更多信息·不做决定 |
| ⚡ Act | 立刻执行 | 纠结·拖延·反复 |

**危机中最大的错误不是做了错误决定——是不做决定。**

因为不做决定本身就是一个决定——你决定把方向盘交给别人。交给命运，交给时机，交给"等一切好了再说"。

## 逆境中的三个反直觉法则

### 法则一：越危机，越透明

大多数人的本能是——遇到危机，藏起来。不跟人说。报喜不报忧。

错的。**危机中最值钱的资源不是钱，是信任。而信任只能在透明中建立。**

跟团队说真话。"我们现在有麻烦了。这是具体情况。这是我知道的。这是我还不知道的。我们需要一起想办法。"

跟客户说真话。"出了这个问题。我在修。预计多久。在此期间你可以做什么。"

**每次你透明地面对危机，你都在给信任账户充值。这个账户的利息比任何银行都高。**

### 法则二：不追求"最优解"，追求"够好+可调"

危机中没有最优解。危机 = 信息不完备 + 时间紧迫 + 后果严重。

在这种情况下追求最优解 = 必然分析瘫痪 = 错过窗口。

**正确做法**：快速找到一个"够好"的方案 → 执行 → 根据反馈调整 → 再执行。

| | 最优解思维 | OODA思维 |
|------|----------|---------|
| 速度 | 慢（收集所有信息） | 快（够好就行） |
| 灵活性 | 低（一条路走到黑） | 高（随时调整方向） |
| 结果 | 分析瘫痪 | 越转越准 |

### 法则三：把危机当压力测试

《反脆弱》作者塔勒布说：有些东西被压之后会碎（脆弱）。有些东西被压之后不变（坚韧）。有些东西被压之后更强（反脆弱）。

| 类型 | 危机后的状态 | 例子 |
|------|------------|------|
| 脆弱 | 碎了 | 单点依赖的系统 |
| 坚韧 | 没变 | 有备份的系统 |
| **反脆弱** | **更强了** | **每次危机后升级的系统** |

**你要建的，不是一个"能扛住危机"的系统。是一个"每次危机后都比之前更强"的系统。**

怎么建？每次危机后，问三个问题：
1. 这次打穿我的弱点是什么？ → 补上。
2. 我在危机中发现了什么新机会？ → 抓住。
3. 下次同样的情况发生，我应该在什么位置？ → 提前布局。

## 景一的冶炼

你现在在做的 72 支柱知识库——从危机应对的角度看，它是什么？

**它是你的反脆弱系统。**

经济好 → 知识库帮你接更多客户。经济不好 → 知识库帮你吸引免费流量。
你身体健康 → 每天冶炼一篇文章。你无法工作 → 自动化管道继续冶炼。
你在 → 你是景一。你不在 → 72 支柱还在 GitHub 上。

**一个反脆弱的人，不是不会遇到危机。是每一次危机之后——他的系统都比之前更强。**
